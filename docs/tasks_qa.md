# QA Stories & Test Cases

## Epic: US-1 Создание упражнения

**Цель:** Проверить API и UI добавления кастомных упражнений на соответствие AC-1, AC-2, AC-3.

### API Тестирование (Postman / cURL)
1. **TC-1.1 (Positive): Успешное создание**
   * *Steps:* `POST /api/exercises` с `{ "name": "Бег" }` и `X-User-Id: test_user`.
   * *Expected:* HTTP 201. В БД появилась запись с `is_custom = true` и `user_id = 'test_user'`.
2. **TC-1.2 (Negative): Пустое имя (AC-3)**
   * *Steps:* `POST /api/exercises` с `{ "name": "" }` или `{"name": "   "}`.
   * *Expected:* HTTP 400 Bad Request. Запись не создана.
3. **TC-1.3 (Negative): Дубликат имени**
   * *Steps:* Создать "Бег", затем повторно отправить `POST /api/exercises` с `{ "name": "Бег" }` и тем же `X-User-Id`.
   * *Expected:* HTTP 409 Conflict (или 400 с понятным текстом). Ошибка уровня БД перехвачена бэкендом.

### UI/UX Тестирование (Браузер)
1. **TC-1.4 (Positive): Переключение UI (AC-1)**
   * *Steps:* Открыть форму создания челленджа. Выбрать в селекте «Добавить свое...».
   * *Expected:* Появляется текстовое поле для ввода названия. При выборе другого пункта из селекта текстовое поле снова скрывается.
2. **TC-1.5 (Positive): Успешный флоу добавления (AC-2)**
   * *Steps:* Выбрать «Добавить свое», ввести «Планка», нажать Сохранить.
   * *Expected:* Отправляется запрос. Интерфейс обновляется: «Планка» появляется в селекте и автоматически выбирается.
3. **TC-1.6 (Negative): Валидация пустого ввода (UI -> API)**
   * *Steps:* Выбрать «Добавить свое», оставить поле пустым, нажать Сохранить.
   * *Expected:* Отправляется запрос (или блокируется на фронте). Если запрос ушел, фронтенд корректно обрабатывает 400 ошибку от сети и показывает красное сообщение об ошибке, не крашась.
